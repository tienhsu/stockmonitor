## ADDED Requirements

### Requirement: 悬浮窗始终置顶 (Always On Top Overlay)
系统必须提供一个默认置顶于其他应用程序窗口之上的“悬浮窗口”，确保用户在操作其他软件时仍能看到。

#### Scenario: 验证置顶功能
- **WHEN** 用户打开其他应用程序（例如浏览器）且悬浮窗处于显示状态
- **THEN** 悬浮窗必须保持在其他应用程序窗口的上方可见

### Requirement: 隐秘模式外观 (Stealth Mode Appearance)
悬浮窗口通常不应显示标准的窗口装饰（标题栏、边框、任务栏图标），以最小化视觉占用。

#### Scenario: 无窗口装饰
- **WHEN** 悬浮窗口显示时
- **THEN** 它必须不显示标题栏、窗口控制按钮（最小化/最大化/关闭）或窗口边框

#### Scenario: 任务栏/Dock 隐藏
- **WHEN** 悬浮窗口处于活动状态
- **THEN** 它不得出现在 Windows 任务栏或 macOS Dock 中（可配置）

### Requirement: 窗口交互 (Window Interaction)
系统必须允许用户重新定位窗口并访问上下文选项。

#### Scenario: 拖拽内容
- **WHEN** 用户在窗口背景的任意位置按住鼠标左键
- **THEN** 拖动鼠标必须移动屏幕上的窗口

#### Scenario: 右键菜单
- **WHEN** 用户在悬浮窗口上右键单击
- **THEN** 必须出现一个上下文菜单，包含选项：“刷新”、“设置”、“隐藏”、“退出”

### Requirement: 窗口透明度控制 (Window Opacity Control)
系统必须支持调节悬浮窗口的透明度，使其能更好地融入桌面背景，降低视觉干扰。

#### Scenario: 透明度生效
- **WHEN** 用户在设置中将透明度调整为 50%
- **THEN** 悬浮窗口必须立即以半透明方式渲染

#### Scenario: 透明度边界
- **WHEN** 透明度设置为最低值（10%）
- **THEN** 窗口仍然必须可见（不可完全隐形）

### Requirement: 鼠标穿透 (Click-Through Mode)
系统必须提供可选的鼠标穿透模式，开启后鼠标点击将穿透悬浮窗口到达下方应用，防止误触。

#### Scenario: 开启穿透模式
- **WHEN** 用户在设置中开启"鼠标穿透"选项
- **THEN** 对悬浮窗口的鼠标点击必须穿透到下方应用窗口 **且** 窗口仅通过快捷键或托盘控制显隐

#### Scenario: 关闭穿透模式
- **WHEN** 用户关闭"鼠标穿透"选项
- **THEN** 悬浮窗口必须恢复正常的鼠标交互（拖拽、右键菜单）
